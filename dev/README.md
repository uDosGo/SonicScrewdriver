# `sonic-family` Dev Workspace

Use `dev` for compact, forward-looking repo planning and current technical notes.

Keep these lanes small:

- `dev/requests/` for a small active index
- `dev/submissions/` for a compact completed summary
- `dev/notes/` for current repo-owned technical notes
- `dev/rounds/` only while a live round is still open

Do not keep long historical request, submission, or duplicate note piles here.
